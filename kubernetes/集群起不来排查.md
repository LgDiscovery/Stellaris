#### 步骤 2：解决`pause`镜像拉取失败问题

K8s 必须依赖`pause`镜像启动 Pod，需手动导入或替换为国内源镜像：

1. **本地导入`pause`镜像**（优先推荐，绕开网络问题）：
   * 在能联网的机器拉取`pause`镜像并导出：
     bash

     运行

     ```
     docker pull registry.aliyuncs.com/google_containers/pause:3.10.1
     docker save -o pause.tar registry.aliyuncs.com/google_containers/pause:3.10.1
     ```
   * 将`pause.tar`传到 master 节点，导入到 containerd：
     bash

     运行

     ```
     ctr -n k8s.io images import /opt/pause.tar
     ```
   * 给镜像打 K8s 默认的标签（让 K8s 识别）：
     bash

     运行

     ```
     ctr -n k8s.io images tag registry.aliyuncs.com/google_containers/pause:3.10.1 registry.k8s.io/pause:3.10.1
     ```

### ✅ 正确修改方法

你现在的 kubelet 启动参数里，`--pod-infra-container-image=registry.aliyuncs.com/google_containers/pause:3.9` 是从 **`/var/lib/kubelet/kubeadm-flags.env`** 这个文件里加载的（因为配置里有 `EnvironmentFile=-/var/lib/kubelet/kubeadm-flags.env`），所以要改这个文件。

---

### 🛠️ 步骤 1：编辑 kubeadm-flags.env

bash

运行

```
nano /var/lib/kubelet/kubeadm-flags.env
```

找到里面的 `--pod-infra-container-image` 这一行，把 `pause:3.9` 改成 `pause:3.10.1`：

env

```
KUBELET_KUBEADM_ARGS="--pod-infra-container-image=registry.k8s.io/pause:3.10.1 --其他参数..."
```

（也可以保留阿里云仓库名，只要版本改成 `3.10.1` 即可）

---

### 🛠️ 步骤 2：重载配置并重启 kubelet

bash

运行

```
systemctl daemon-reload
systemctl restart kubelet
```

---

### 🛠️ 步骤 3：验证修改生效

bash

运行

```
ps aux | grep kubelet | grep pod-infra-container-image
```

如果输出变成：

plaintext

```
--pod-infra-container-image=registry.k8s.io/pause:3.10.1
```

就说明修改成功了 ✅

---

### 🛠️ 步骤 4：导入并保留 pause:3.10.1

现在 kubelet 认这个版本了，导入后就不会被删除：

bash

运行

```
# 导入镜像
ctr -n k8s.io images import pause.tar
# 打标签（如果需要）
ctr -n k8s.io images tag registry.aliyuncs.com/google_containers/pause:3.10.1 registry.k8s.io/pause:3.10.1
# 验证
crictl images | grep pause
```

---

### 💡 为什么要改这个文件？

* `kubeadm-flags.env` 是 kubeadm 生成的动态参数文件，优先级比 `KUBELET_EXTRA_ARGS` 更高。
* 你之前改的 `config.yaml` 没生效，就是因为这里的启动参数覆盖了它。
* 改完这个文件后，kubelet 会把 `pause:3.10.1` 当成**官方沙箱镜像**，再也不会自动清理了。
