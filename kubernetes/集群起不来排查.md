#### 查看kubelet日志

journalctl -u kubelet -f

#### 步骤 2：解决`pause`镜像拉取失败问题

K8s 必须依赖`pause`镜像启动 Pod，需手动导入或替换为国内源镜像：

1. **本地导入`pause`镜像**（优先推荐，绕开网络问题）：
   * 在能联网的机器拉取`pause`镜像并导出：
     bash

     运行

     ```
     docker pull registry.aliyuncs.com/google_containers/pause:3.10.1
     docker save -o pause.tar registry.aliyuncs.com/google_containers/pause:3.10.1
     ```
   * 将`pause.tar`传到 master 节点，导入到 containerd：
     bash

     运行

     ```
     ctr -n k8s.io images import /opt/pause.tar
     ```
   * 给镜像打 K8s 默认的标签（让 K8s 识别）：
     bash

     运行

     ```
     ctr -n k8s.io images tag registry.aliyuncs.com/google_containers/pause:3.10.1 registry.k8s.io/pause:3.10.1
     ```

#### 步骤 3：重启 containerd 和 kubelet，触发静态 Pod 启动

bash

运行

```
# 重启containerd
systemctl restart containerd
# 重启kubelet，触发静态Pod重建
systemctl restart kubelet
```

#### 步骤 4：验证容器是否启动

bash

运行

```
# 查看containerd中的容器
crictl ps
# 查看K8s核心组件的Pod状态
kubectl get pods -n kube-system
```
