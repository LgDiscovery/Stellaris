# 一、数据漂移、概念漂移的定义

### 数据漂移（Data Drift）

工程视角下，指**模型输入的特征数据分布**在训练阶段和线上推理阶段发生了**显著变化**，是**输入侧**的分布偏移。简单说就是模型 “吃的料” 和当初训练时不一样了，特征的统计属性（均值、方差、取值范围、类别占比等）出现明显差异。

### 概念漂移（Concept Drift）

指**输入特征与预测目标之间的关联关系**发生了变化，是**映射关系**的偏移，即模型学习的 “数据规律 / 业务逻辑” 失效了。简单说就是当初的 “特征→结果” 的对应关系变了，即使输入数据分布没变化，模型预测也会不准。

（补充：概念漂移常和数据漂移伴随出现，数据漂移是表象，概念漂移可能是本质原因，二者都会导致模型退化。）

# 二、工程视角下数据漂移对模型的影响 + 如何发现漂移

## （一）数据漂移对模型的核心影响

1. **模型预测精度大幅下降**：模型是基于训练数据的分布学习规律，输入数据分布偏移后，模型对新数据的拟合度骤降，出现准确率、召回率下跌，误判 / 漏判增多等问题；
2. **线上推理结果不可靠**：漂移会导致模型输出异常值、不符合业务实际的结果，比如风控模型把正常用户判定为高风险，推荐模型推荐与用户偏好无关的内容；
3. **模型泛化能力彻底失效**：模型无法适配新的数据分布，从 “通用模型” 变成 “仅适配训练数据的专属模型”，失去业务落地价值；
4. **工程运维成本陡增**：漂移若未及时发现，会引发线上业务问题（如推荐转化暴跌、风控误判投诉），后续需重新采集数据、做特征工程、训练模型，增加 MLOps 全流程的工作量。

## （二）工程视角下如何发现数据漂移

核心思路：**对训练集（基准分布）和线上推理的实时 / 离线特征数据做「分布对比 + 监控告警」**，落地以**自动化监控**为主，人工核验为辅，核心方法如下：

1. **统计指标监控**
   对数值型特征，实时监控均值、方差、分位数、极值的变化；对分类型特征，监控类别占比、Top 类别变化。若线上指标与训练集基准指标的偏差超过预设阈值（如均值偏差 ±20%、类别占比偏差 ±15%），触发漂移告警。
2. **分布相似度检验**
   用统计学方法量化训练集和线上数据的分布差异，如数值型特征用**KS 检验、AD 检验**，分类型特征用**卡方检验、Jaccard 相似度**，若检验结果的 p 值小于预设阈值（如 0.05），判定为存在显著漂移。
3. **特征稳定性指标（PSI）监控**
   工程中最常用的漂移检测指标，计算特征在训练集和线上集的分箱占比差异，**PSI 值越大，漂移越严重**（行业通用阈值：PSI<0.1 无漂移，0.1≤PSI<0.25 轻度漂移，PSI≥0.25 重度漂移），可对所有特征做 PSI 批量计算和告警。
4. **业务指标联动监控**
   模型的业务指标是漂移的 “最终体现”，若线上业务指标（如推荐点击率、风控精准率、预测准确率）出现无理由的持续下跌，反向排查是否存在数据漂移，是工程中最直观的漂移信号。
5. **离线复盘与根因分析**
   对告警的漂移特征，做离线的分布可视化（如直方图、饼图、折线图），对比训练集和线上集的特征分布形态，定位漂移的具体特征、漂移开始的时间节点，结合业务场景分析漂移原因（如用户行为变化、数据采集链路故障）。

# 三、数据漂移的实际场景举例

### 场景：外卖平台**订单预测模型**的时间型 + 数值型特征数据漂移

1. **模型背景**：外卖平台训练订单预测模型，用于预估各商圈每小时的订单量，指导骑手调度、商家备货，训练数据为**2026 年 1-2 月**的商圈特征数据，核心特征包括「小时订单量（数值型）」「工作日 / 周末占比（分类型）」「天气类型占比（分类型）」「人均下单金额（数值型）」。
2. **漂移发生**：进入**2026 年 3 月**后，模型线上预测的订单量与实际订单量偏差持续超过 30%，经检测发现核心特征出现显著数据漂移：
   * 数值型特征「小时订单量」：训练集（1-2 月）晚 6-8 点峰值订单量均值为 500 单 / 商圈，3 月线上数据该时段均值骤降至 300 单 / 商圈，方差从 80 变为 150，分布发生大幅偏移；
   * 分类型特征「天气类型占比」：训练集（1-2 月）晴天占比 70%、雨天占比 20%，3 月当地进入梅雨季节，线上数据雨天占比升至 60%、晴天占比仅 30%，类别占比完全偏移；
   * 数值型特征「人均下单金额」：训练集均值 35 元，3 月因平台上线 “轻食特惠活动”，线上数据均值降至 22 元，分布偏移明显。
3. **漂移原因**：季节变化导致天气特征分布改变，平台营销活动改变用户下单金额和下单行为，最终引发模型输入特征的分布漂移，导致订单预测模型的精度大幅下降，出现骑手冗余 / 不足、商家备货过多 / 缺货的业务问题。

（补充：该场景中若伴随**概念漂移**，比如 “雨天” 原本对应订单量上升（1-2 月冬季，雨天用户更愿点外卖），3 月梅雨季节持续降雨，用户出现 “点外卖疲劳”，雨天反而订单量下降，这就是「天气特征→订单量」的映射关系发生变化，属于概念漂移。）
