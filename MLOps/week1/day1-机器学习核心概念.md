# 一、机器学习核心概念（工程视角，无算法推导）

工程视角下，机器学习可理解为**数据驱动的规律拟合工具**，核心是让计算机从历史数据中学习规律，再用规律对新数据做预测 / 判断，无需人工编写固定逻辑，落地核心围绕 5 个关键环节，无复杂算法推导，仅需掌握工程落地的核心认知：

1. **数据层**：机器学习的核心原料，分为训练集（教模型学规律）、验证集（调优模型参数）、测试集（检验模型最终效果），工程中重点关注数据的清洗、标注、归一化，保证数据质量是模型效果的基础。
2. **模型层**：是承载 “学习规律” 的载体，本质是预设的计算框架（如分类用的 CNN、回归用的线性模型），工程中无需深究框架的数学推导，只需根据业务场景（分类 / 回归 / 聚类）选择合适的模型类型。
3. **训练过程**：把训练数据输入模型，让模型通过迭代调整自身参数，逐步拟合数据中的规律，工程中核心关注训练环境搭建、训练过程监控（防止过拟合 / 欠拟合）、训练资源调度。
4. **推理 / 部署**：把训练好的模型放到生产环境，接收新数据并输出结果，是 MLOps 的核心环节，工程中重点关注模型的轻量化、推理速度、硬件适配（CPU/GPU/ 端侧设备）。
5. **模型评估**：用测试集验证模型效果，核心看业务相关指标（如分类任务的准确率、回归任务的均方误差），工程中需保证评估指标与实际业务场景对齐，而非单纯追求理论指标。

# 二、模型的本质（工程视角）

模型的本质是**经过训练后，存储了数据中核心规律的「参数集合 + 计算逻辑」的封装体**。

简单来说，训练的过程就是模型不断调整自身参数的过程，训练完成后，这些参数和模型的计算逻辑被一起保存为文件，就是我们拿到的模型文件；当输入新数据时，模型通过固定的计算逻辑，结合已学习的参数，就能输出对应的预测 / 判断结果，这就是模型的核心作用。

# 三、三种常见模型格式（.pth/.onnx/.ckpt）的区别与适用场景

表格


| 格式  | 核心定义                                  | 关键特点                                                                | 核心适用场景                                                                                        |
| ----- | ----------------------------------------- | ----------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| .pth  | PyTorch 框架专属模型格式                  | 直接保存 PyTorch 模型的参数 / 完整模型，与 PyTorch 强绑定，读写速度快   | PyTorch 框架下的**模型训练、调优、本地测试**，研发阶段高频使用                                      |
| .ckpt | 模型检查点文件（主流 PyTorch/TensorFlow） | 不仅保存模型参数，还可保存训练过程的优化器状态、epoch、损失值等训练信息 | 模型**断点续训**（训练中断后继续训练）、训练过程复现、研发阶段的模型保存                            |
| .onnx | 开放的跨框架模型中间格式                  | 与训练框架解耦（PyTorch/TensorFlow/TF Lite 均可转换），做了轻量化优化   | 工业界**模型跨框架部署、端侧 / 边缘侧部署**（如手机、嵌入式设备）、模型推理加速，MLOps 落地核心格式 |
