# 一、工程视角下特征工程基础

### 1. 特征是什么

特征是从**原始业务数据**中提取 / 加工后，能被模型识别、用于学习规律的**结构化信息**，是模型的 “输入素材”。

比如用户消费数据中，“消费金额”“消费频次”“是否会员”，商品数据中 “价格”“品类”“销量”，都是特征；原始数据是杂乱的日志 / 记录，特征是对其的工程化转化。

### 2. 特征的核心作用

1. **筛选有效信息**：剔除原始数据中的噪声、冗余内容，只保留对模型预测 / 分类有帮助的信息，减少模型学习成本；
2. **统一输入标准**：将不同类型、不同量纲的原始数据转化为模型可处理的格式，让模型能正常进行计算；
3. **提升模型效果**：优质的特征能让模型快速捕捉业务规律，大幅提升预测准确率、泛化能力，比单纯调模型参数更高效（工程界有 “数据和特征决定模型上限，算法只是逼近这个上限” 的说法）；
4. **适配业务场景**：结合业务逻辑加工特征，让模型学习的规律更贴合实际业务需求，避免模型 “学偏”。

# 二、特征预处理的简单流程（工程落地版）

特征预处理是**统一特征格式、消除数据缺陷、提升特征有效性**的标准化步骤，所有特征需先经过预处理，才能输入模型训练 / 推理，核心流程为：

1. **特征采集与清洗**：从业务数据库 / 日志中提取特征→过滤缺失值、异常值（如远超出正常范围的极端数）、重复值→补全必要的缺失信息（如用均值 / 中位数填充）；
2. **特征类型识别**：区分特征的类型（数值型、分类型、时间型等），按类型做针对性处理；
3. **特征转换 / 归一化**：对分类型特征做编码、数值型特征做缩放 / 归一化，消除量纲和类型差异；
4. **特征筛选**：剔除冗余特征、低相关性特征（如与预测目标无关的特征），减少特征维度，降低模型计算量；
5. **特征标准化输出**：将处理后的特征整理为模型可直接读取的结构化格式（如数组、矩阵），接入训练 / 推理的数据管道。

# 三、3类 常见特征类型 + 预处理核心目的

### 1. 数值型特征

**定义**：用数字表示、具有大小 / 量化意义的特征，如消费金额、身高、销量、点击量、年龄（连续 / 离散数字均属于）。

**预处理核心目的**：

① 消除**量纲差异**：比如 “消费金额（元，0-10000）” 和 “消费频次（次，0-100）” 量纲不同，直接输入会让模型侧重数值大的特征，需做归一化 / 标准化（如缩放到 0-1 区间、转化为均值 0 方差 1 的标准值）；

② 抑制**异常值影响**：通过截断、平滑等方式，避免极端值干扰模型学习的规律；

③ 离散化（可选）：将连续数值拆分为区间（如年龄分为 18 岁以下、18-30 岁等），让模型更易捕捉非线性规律。

### 2. 分类型特征

**定义**：表示类别、无大小量化意义的特征，如性别（男 / 女）、商品品类（食品 / 服饰 / 家电）、用户等级（VIP / 普通）、是否点击（是 / 否）（无序 / 有序类别均属于）。

**预处理核心目的**：

① **实现数值化转化**：模型只能处理数字，需将文字 / 标签类的类别特征编码为数字（如独热编码、标签编码）；

② 避免**虚假数值关系**：若直接用 1、2、3 表示 “食品、服饰、家电”，模型会误判为 “服饰 > 食品、家电 > 服饰” 的大小关系，需通过编码消除这种虚假逻辑；

③ 简化**高基数类别**：若类别数量过多（如全国城市），需做归并（如按省份合并），减少特征维度。

### 3. 时间型特征

**定义**：表示时间 / 时序的原始特征，如交易时间、点击时间、注册时间（如 2026-03-14 18:00、2026/03/01），是工程中高频的复合特征类型。

**预处理核心目的**：

① **提取有效时序信息**：将原始时间戳 / 日期，加工为模型可利用的特征（如小时、星期、月份、季度、是否节假日、距当前时间的间隔天数），挖掘时间规律（如商品消费高峰在晚 8 点、周末）；

② **消除时间格式差异**：统一不同的时间书写格式（如 yyyy-mm-dd、mm/dd/yyyy），转化为结构化的时间特征；

③ **数值化时序特征**：将加工后的时间特征（如星期 1-7、月份 1-12）转化为数字，适配模型计算；

④ 平滑**时序噪声**：对时序数据做滑动平均，消除短期波动，让模型捕捉更稳定的时间规律。
